================================================================================

STATUS: Complete

A full technical development report was generated documenting the entire
project as it existed at the start of Day 4.

File created:
  PM-Updates.txt

Report covers:
  - Project overview and user flow
  - Complete folder structure with explanations
  - Set data system (TXT format, JSON structure, manifest discovery)
  - Pack generation system (rarity pools, slot odds, finish assignment)
  - Pack opening pipeline end-to-end (all 6 steps)
  - Data flow diagrams (startup, set load, pack open, collection)
  - TXT → JSON generation script technical details
  - Local development environment setup
  - Build status
  - Known limitations including the active Pinsir rarity bug

Key findings documented:

  ACTIVE BUG — Rarity String Mismatch
  jsonSetLoader.ts expects "Holo Rare" but JSON files store "Rare Holo".
  All unmatched rarities default to Common. This silently places holo rares
  (e.g., Pinsir #009 Jungle, all Base Set holos) into the common pool.
  Affects: base-set.json and jungle.json. Fix is queued for Day 4.

  LIMITATION — Pack Config Not Set-Specific
  All sets use DEFAULT_PACK_CONFIG (Sword & Shield era structure).
  Classic sets (Base Set, Jungle) had different pack layouts.
  Fix is queued for Day 4.

  LIMITATION — No Per-Card JSON Schema Validation
  Card objects are not validated at conversion or load time.
  Malformed entries produce silent incorrect mappings.

--- WORKFLOW GOING FORWARD ---

  Normal development:           npm run dev
  Open in native window (dev):  npm run tauri:dev
  Build new installer:          npm run tauri:build

  Dev server (port 3000) and Tauri dev window can run simultaneously.
  The installed app is completely independent of the source code —
  changes to src/ do not affect the installed version until a new
  tauri:build is run and the installer is re-run.
