2. **Storage Efficiency Issue - Unnecessary Data Persisted**

   - Impact: Reduces localStorage size, cleaner data persistence
   - Location: hooks/useSetManager.ts saveToStorage()

3. **Debugging Enhancement**

   - Helps verify persistence working correctly

VERIFICATION OF EXISTING LOGIC:

**Collection Update Flow:**
1. Pack opened with 11 cards
2. Each card checked against collection (number + finish match)
3. If exists: increment count
4. If new: add with count = 1
5. Collection sorted by card number
6. SavedSets state updated immutably
7. saveToStorage() called immediately
8. localStorage updated synchronously

**Persistence Flow:**
1. On app load: loadFromStorage() runs during useState initialization
2. Parses JSON from localStorage
3. Rebuilds rarityPools for each set
4. Returns complete SavedSet array with collections intact

**Collection Item Structure:**
```typescript
{
  number: string,    // Card number (e.g., "25")
  name: string,      // Card name
  rarity: Rarity,    // Common, Uncommon, Rare, V, VMAX, Secret Rare
  finish: FinishType, // Standard, Reverse Holo, Holo, Ultra Rare, Secret Rare
  count: number      // How many owned
}
```

Each card number + finish combination tracked separately (correct for collecting)
